  • The unzipped repository package.
  • GitHub Desktop recommended for easiest upload.
  • Optional: a trusted reviewer account.

Already configured

  • .github/CODEOWNERS uses active @MontrealAI rules.
  • MAINTAINERS.md identifies @MontrealAI.
  • Repository readiness checks expect MontrealAI ownership.

Safe default

Start with a private repository until the first CI run is green and repository protection is enabled.

Fast path: click-by-click

1

Create the repository under MontrealAI

In GitHub, click +, then New repository. Owner: MontrealAI. Name: goalos-signoff-pro. Visibility: Private for first setup.

2

Upload the package

Use GitHub Desktop if possible. Clone the empty repository, copy every file from this package into that folder, commit with Initial GoalOS Signoff Pro repository, and push.

GitHub Desktop is safer for hidden folders such as .github.

3

Confirm CODEOWNERS

Open .github/CODEOWNERS. It should contain active @MontrealAI rules. No owner-neutral setup step is required.

4

Wait for GitHub Actions

Go to Actions. The CI and repository-readiness workflows should run automatically after upload. Your first required result is green.

5

Protect the repository

Open Settings. Follow docs/repository/REPOSITORY_SETTINGS_CHECKLIST.md. Require pull requests, status checks, CODEOWNER review, and block force pushes.

6

Add deployment secrets only later

Do not add production keys until the repository is protected. Never commit secrets into files.

7

Run the first launch test

Use docs/PRO_V1_1_LAUNCH_PLAYBOOK.md and docs/LAUNCH_ACCEPTANCE_TEST.md. Do not invite real customers until the test passes.
