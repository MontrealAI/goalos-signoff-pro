# Proof-Gated Coordination User Guide

## What this demo means

The demo shows a large synthetic multi-agent system becoming a proof-governed institution.

It does not ask for your data. It does not connect to a wallet. It does not move value.

## What to click

1. Open `proof-gated-coordination.html`.
2. Read the core law: maximum coordination is not maximum autonomy.
3. Open `coordination-lab.html`.
4. Click `Launch coordination cycle`.
5. Watch the coordinator, planner, builder, evidence scout, validator, risk sentinel, Chronicle, and human gate advance.
6. Open `evidence-docket-demo.html` to inspect the proof room.
7. Open `verify.html` to verify the built-in synthetic receipt.

## What to tell users

GoalOS does not optimize for agent chatter. It optimizes for verified work.

The loop is:

```text
Objective → Mission Contract → Agent / Job Allocation → Tool Execution → Proof Bundle → Evidence Docket → Verifier Mesh → Governed Decision State → Settlement Signal → Chronicle → Capability Package → Harder Future Mission
```

## Safety posture

Every page is browser-local and public-safe:

```text
No forms
No uploads
No cookies
No analytics
No wallets
No payments
No personal or confidential data
```
