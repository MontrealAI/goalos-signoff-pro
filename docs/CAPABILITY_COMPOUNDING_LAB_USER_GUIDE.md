# GoalOS Capability Compounding Lab — User Guide

## What this demo shows

Most AI systems produce outputs. GoalOS asks a harder question:

```text
Which outputs deserve to affect the next mission?
```

The Capability Compounding Lab shows the answer:

```text
Only accepted proof becomes reusable capability.
```

## How to run it

Open:

```text
https://montrealai.github.io/goalos-signoff-pro/capability-compounding-lab.html
```

Click:

```text
Run compounding cycle
```

The browser runs three public-safe synthetic missions:

```text
M1 — first governed decision
M2 — reuse the proof pattern
M3 — harder mission with memory
```

The lab shows verified work increasing and proof debt decreasing.

## What to look for

The important moment is not the score. It is the gate:

```text
Proof → Selection → Chronicle → Capability
```

GoalOS does not let unsupported output become institutional memory. It records only proof-carrying results.

## What you can download

The page can download a public-safe JSON demo bundle. It contains:

```text
mission series
proof bundle hashes
selection certificates
capability packages
Chronicle entries
synthetic receipt
```

No user data is collected. No data leaves the browser.
