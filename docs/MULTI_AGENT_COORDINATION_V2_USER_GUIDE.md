# Multi-Agent Coordination Demo — User Guide

Open the demo page:

```text
https://montrealai.github.io/goalos-signoff-pro/coordination-theatre.html
```

## What to do

Click one scenario:

```text
AI research acceptance
Automation delivery review
Grant milestone proof
Vendor proof review
```

Then click:

```text
Launch coordination cycle
```

The page will run a browser-local simulation of a proof-governed institution.

## What you are seeing

The demo shows how GoalOS turns a large agent population into a governed work system:

```text
Coordinator          selects roles and budget
Planner              creates role contracts
Evidence Scout       maps claims and support
Builder              packages the deliverable
Red-Team Critic      surfaces failure modes
Validator            checks proof and replay
Risk Sentinel        protects boundaries
Chronicle            records accepted experience
```

## What this demo is not

It is not a live agent run. It is not a wallet. It is not a payment. It does not upload data. It does not collect data. It does not move value. It is a browser-local education and demonstration layer.

## What to remember

The point is simple:

```text
Not more agents.
Better institutions.
```

A large multi-agent system coordinates to optimal effect when proof governs what can be accepted, remembered, reused, and economically consequential.
