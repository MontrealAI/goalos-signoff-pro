# Until-DONE Mission Control Lab — User Guide

The Until-DONE Lab shows the difference between ordinary AI output and GoalOS proof-to-action work.

Ordinary AI can produce a persuasive answer. GoalOS keeps working until the acceptance package exists:

```text
Mission Contract
Claims Matrix
Source Provenance
Contradiction Register
Evidence Docket
Verifier Report
Risk Ledger
Governed Decision State
Action Graph
Chronicle Entry
Claim Boundary Pass
QA Pass
```

## How to use it

1. Open `until-done-lab.html`.
2. Choose a scenario.
3. Click **Run until DONE**.
4. Watch proof debt decrease.
5. Inspect the generated tabs: Mission Contract, Proof Debt Ledger, Decision State, Action Graph, Chronicle, Certificate.
6. Download the synthetic demo bundle.

## What to look for

The important moment is the loop:

```text
Evidence Gap → return to proof work
```

GoalOS does not accept a mission simply because the output is fluent. If evidence, contradiction handling, risk state, or rollback path is missing, the state machine returns to work.

## What this demonstrates

```text
A report can be done because it sounds complete.
A GoalOS mission is DONE only when the proof package is complete.
```

## What it does not do

The lab does not collect data, upload files, connect wallets, move value, certify production work, claim external audit, or activate settlement.
