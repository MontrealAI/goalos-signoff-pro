# Release and Tagging Guide

## Version naming

Use semantic versioning for the product:

```text
v1.1.0-private-beta.1
v1.1.0-private-beta.2
v1.1.0
```

## Release checklist

Before creating a GitHub release:

```text
[ ] CI green on main
[ ] No known critical or high security issues
[ ] Changelog updated
[ ] Package verification green
[ ] Pilot launch boundary still accurate
[ ] Deployment notes updated
[ ] Rollback plan documented
```

## Recommended first release

```text
v1.1.0-private-beta.1
```

Release title:

```text
GoalOS Signoff Pro v1.1 — Private Beta 1
```

Release description:

```text
Mainstream Signoff workflow with signed Mission Receipts, evidence assistant, change requests, and optional verified-receipt path. No escrow, custody, AGIALPHA staking, or Mainnet settlement.
```
