# First 30 Days Plan

## Week 1 — Repository and deployment readiness

- Create GitHub repository.
- Upload package.
- Get CI green.
- Protect main branch.
- Set up Supabase project.
- Run database migrations.
- Generate receipt signing keys.
- Deploy private beta app.

## Week 2 — Internal dry run

- Create one test client.
- Create one test builder.
- Create one test reviewer.
- Run one complete Signoff.
- Export receipt PDF and JSON.
- Verify receipt offline.
- Fix onboarding friction.

## Week 3 — Trusted pilot

- Invite 3–5 trusted users.
- Use one of the five templates.
- Collect feedback.
- Record failed or confusing steps.

## Week 4 — Product decision

- Review pilot metrics.
- Decide whether to continue, simplify, or reposition.
- Publish a private-beta learnings memo.
- Prepare v1.1.1 improvements.

## Success threshold

```text
10 Signoffs attempted
7 completed
3 users say they would use it again
0 lost evidence
0 broken receipts
0 security incidents
```
