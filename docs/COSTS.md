# Operating costs

## What Phase 1 does not cost

There is no:

- Ethereum gas
- ETH requirement
- AGIALPHA requirement
- wallet service
- smart-contract transaction
- escrow capital requirement

## Normal software costs

Actual prices change by provider and usage. Check each provider’s current pricing before launch.

| Category | Private-beta expectation | Cost driver |
|---|---|---|
| Web hosting | Often low at pilot scale | Traffic, server functions, commercial plan |
| Database and authentication | Often low at pilot scale | Active users, database size, backups |
| Private file storage | Usage-based | File volume, downloads, retention |
| Email | Optional | Invitation and magic-link volume |
| Domain | Annual | Registrar and domain choice |
| Monitoring and backups | Recommended | Retention, alerting, recovery objectives |
| Support | Human operating cost | Users, incidents, and reviews |

## A sensible pilot budget

For a small invitation-only pilot with modest files, plan for a low monthly infrastructure budget plus a domain. Keep provider spending alerts and limits enabled.

The largest serious-production expenses are not raw hosting. They are security review, monitoring, backups, privacy/legal work, support, incident response, and eventually independent audits.

## Cost-control settings

- Limit each file to 25 MB.
- Begin with 5–20 invited users.
- Use private receipts by default.
- Set retention periods.
- Enable provider budget alerts.
- Avoid AI APIs in Phase 1 unless a paid check provides clear customer value.
- Add email only after manual invitation links become inconvenient.
