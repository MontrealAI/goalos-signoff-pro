# Mission Foundry Lab — User Guide

Open:

```text
https://montrealai.github.io/goalos-signoff-pro/mission-foundry-lab.html
```

Then click:

```text
Run foundry gate
```

The page shows how GoalOS turns accepted proof into the next harder mission.

## The core idea

A normal task generator can create endless ideas. GoalOS does something stricter: it only admits mission seeds that are useful, learnable, proofable, validator-bounded, replayable, challengeable, and safe to inspect.

## What you will see

The browser evaluates four synthetic mission seeds:

```text
M0 — Narrative moonshot
Rejected: interesting language, no proof path.

M1 — Promising task idea
Held: useful, but validator requirements are incomplete.

M2 — High-impact escalation
Quarantined: impact is high, but risk boundary is insufficient.

M3 — Proof-gated harder mission
Admitted: bounded, replayable, challengeable, and useful.
```

## What you can download

Click:

```text
Download demo bundle
```

The browser downloads a public-safe JSON bundle. It contains a mission curriculum, a seed certificate, a filter report, and a quarantine ledger.

## What the demo does not do

It does not request user data. It does not upload anything. It does not connect a wallet. It does not move value. It does not claim external audit, production certification, live settlement, AGI attainment or ASI attainment.
