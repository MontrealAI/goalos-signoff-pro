# GoalOS Signoff Pro v47 — Highest Command Experience

## What this adds

A separate, front-and-center command experience for the public website.

The user sees one simple instruction:

> Tell GoalOS what you want.

GoalOS compiles the request into:

1. mission type;
2. acceptance criteria;
3. evidence checklist;
4. reviewer path;
5. relevant 48-contract rail map;
6. recommended next page;
7. synthetic Mission Receipt.

## Why this exists

The site has many valuable labs, demos, contract rails, receipts, and governance pages. v47 makes them easy to use without requiring the visitor to understand the whole system first.

## Public-safe default

The command box runs in the browser by default. It does not upload content, call an external AI provider, connect a wallet, use analytics, move value, or require personal/confidential data.

A future live AI endpoint must be server-side, rate-limited, origin-restricted, route-allowlisted, and separately authorized. Never expose provider API keys in browser JavaScript.
