# Technical Note — Proof-Governed Coordination

The v2 browser experience encodes a coordination target:

- maximize verified work, evidence integrity, capability reuse, coalition stability, and human-review readiness;
- minimize compute cost, safety risk, coordination overhead, unknown externality, and false acceptance;
- require hard gates for proof validity, eval pass, risk bound, rollback readiness, human authority, and claim boundary.

The demo is not a benchmark claim. It is a public-safe interface pattern for explaining the architecture.

The repository verifier blocks public pages containing forms, inputs, textareas, uploads, wallet connection language, analytics/cookie APIs, or unsupported achievement claims.
