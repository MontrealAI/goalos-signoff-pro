# AGIALPHA tokenomics and protocol path

## Product principle

Stable money pays for the work. AGIALPHA secures honesty and protocol usage.

## In this package

AGIALPHA is optional and disabled by default.

`GoalOSSignoffAnchorV1` includes an optional AGIALPHA protocol fee, but the recommended private-beta value is zero.

This allows the product to launch commercially without forcing customers to acquire a token.

## Later AGIALPHA uses

After security and governance gates:

1. Verified receipt protocol fee paid by the operator or enterprise account.
2. Reviewer bond for accountable review.
3. Builder bond for high-stakes work.
4. Dispute/challenge stake.
5. Credential and proof-card issuance fee.
6. Optional work rewards where customers explicitly choose token settlement.

## What not to claim

Do not claim:

- guaranteed yield
- revenue share
- price appreciation
- automatic buyback
- automatic burn
- investment return

The token should be framed as a utility and security asset for the GoalOS work-acceptance protocol.

## Mainstream customer experience

Mainstream customers should be able to use the product without knowing AGIALPHA exists. Token utility can operate at the protocol, operator, reviewer, or enterprise-contract level.
