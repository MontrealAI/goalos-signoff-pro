# Operations Runbook

**Product:** GoalOS Signoff Pro v1.1  
**Audience:** GoalOS team, pilot customers, operators  
**Owner:** GoalOS product and operations  
**Status:** Ready for private beta  
**Version:** 1.1.0  
**Updated:** 2026-06-26T00:05:06Z

> Daily and weekly operating procedures for a small private beta.

---

## Daily checks

- [ ] App loads
- [ ] Login emails deliver
- [ ] New project creation works
- [ ] File upload works
- [ ] Receipt generation works
- [ ] No critical support tickets open
- [ ] No unauthorized-access alerts

## Weekly checks

- [ ] Export pilot metrics
- [ ] Review incomplete Signoffs
- [ ] Review change-request reasons
- [ ] Review billing/invoice status
- [ ] Review failed AI assistant checks
- [ ] Review storage growth
- [ ] Review audit logs

## Support severity

| Severity | Example | Response target |
| --- | --- | --- |
| S1 | Unauthorized access or data exposure | Immediate |
| S2 | Receipt generation broken | Same day |
| S3 | Upload issue for one user | 1 business day |
| S4 | Product question | 2 business days |


## Operating principle

When in doubt, protect customer data and preserve the audit trail before optimizing for speed.
