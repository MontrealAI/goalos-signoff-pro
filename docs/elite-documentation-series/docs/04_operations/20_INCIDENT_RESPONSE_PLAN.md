# Incident Response Plan

**Product:** GoalOS Signoff Pro v1.1  
**Audience:** GoalOS team, pilot customers, operators  
**Owner:** GoalOS product and operations  
**Status:** Ready for private beta  
**Version:** 1.1.0  
**Updated:** 2026-06-26T00:05:06Z

> What to do when something goes wrong.

---

## Incident types

- Unauthorized access
- Evidence loss or corruption
- Receipt signing failure
- Wrong person invited
- AI assistant harmful output
- Billing mistake
- Blockchain anchor mismatch
- Service outage

## First 30 minutes

- [ ] Assign incident lead
- [ ] Stop the bleeding
- [ ] Preserve logs
- [ ] Record timeline
- [ ] Disable affected feature if necessary
- [ ] Notify internal stakeholders
- [ ] Do not delete evidence

## Customer communication template

```text
Subject: GoalOS Signoff incident update

We identified an issue affecting [scope].
What happened: [plain-language summary]
What data was affected: [specific scope]
What we have done: [containment]
What happens next: [next update time]
```

## Post-incident review

- [ ] Root cause identified
- [ ] Customer impact confirmed
- [ ] Controls improved
- [ ] Documentation updated
- [ ] Tests added
- [ ] Owner assigned
- [ ] Deadline set
