# Glossary

**Product:** GoalOS Signoff Pro v1.1  
**Audience:** GoalOS team, pilot customers, operators  
**Owner:** GoalOS product and operations  
**Status:** Ready for private beta  
**Version:** 1.1.0  
**Updated:** 2026-06-26T00:05:06Z

> Plain-language definitions for everyone on the team.

---

| Term | Meaning |
| --- | --- |
| Acceptance criteria | The conditions that define done |
| Builder | The person/team/agent delivering the work |
| Client | The person who needs the work accepted |
| Reviewer | The person who checks evidence and recommends a decision |
| Mission Receipt | Signed record of what was accepted |
| Evidence | Files, links, screenshots, tests, and notes supporting the work |
| Evidence assistant | GoalOS checker that flags missing or weak evidence |
| Verified receipt | Receipt hash optionally anchored to IPFS/Ethereum |
| AGIALPHA | Future protocol token used for bonds, fees, and security, not required in v1.1 |
| Proof-to-payment | Future flow where accepted work can release payment |
