## Reviewer responsibility

You are not proving universal truth. You are assessing whether the submission meets the stated acceptance criteria.

## Recommended decision framework

| Decision | Use when |
| --- | --- |
| Recommend accept | Criteria are satisfied and limitations are reasonable |
| Request changes | Work is close but fixable issues remain |
| Recommend reject | Major criteria are unmet or evidence is insufficient |
| Escalate | Risk, legal, security, or factual uncertainty exceeds reviewer scope |


## Reviewer checklist

- [ ] Confirm the correct brief version
- [ ] Review each criterion
- [ ] Open mapped evidence
- [ ] Check limitations
- [ ] Check AI-use disclosure if required
- [ ] Record unanswered questions
- [ ] Avoid accepting unsupported claims
- [ ] Use clear, actionable change requests

## Reviewer language

Good:

> Criterion 3 is not yet satisfied because the pricing table lacks source links for two competitors.

Poor:

> Needs work.
