> How a consultant, agency, or contractor submits work that can be accepted cleanly.

---

## Your goal

Make it easy for the client to accept the work by mapping deliverables to evidence.

## Submission checklist

- [ ] Read every acceptance criterion
- [ ] Upload the deliverable
- [ ] Attach supporting evidence
- [ ] Map evidence to each criterion
- [ ] Disclose what remains uncertain
- [ ] Explain AI-assisted parts
- [ ] Respond to evidence assistant warnings
- [ ] Submit for review

## What good evidence looks like

| Work type | Strong evidence |
| --- | --- |
| Research | Source list, dates, claims matrix, limitations |
| Automation | Demo video, screenshots, SOP, rollback instructions |
| Software | Tests, commit hash, screenshots, deployment notes |
| Content | Final assets, brand checklist, approval notes |
| Grant milestone | Milestone proof, impact notes, public links |


## How to avoid change requests

- Do not leave criteria unmapped
- Do not hide limitations
- Do not upload final files without context
- Do not claim certainty beyond evidence
- Do not submit a different scope than the brief
