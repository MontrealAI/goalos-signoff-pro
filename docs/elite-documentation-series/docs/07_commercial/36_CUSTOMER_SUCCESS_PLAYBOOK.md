## Onboarding call agenda

| Minute | Topic |
| --- | --- |
| 0–5 | What kind of work they deliver |
| 5–10 | Where acceptance currently breaks down |
| 10–20 | Create first Signoff together |
| 20–30 | Explain evidence and change requests |
| 30–35 | Show receipt and verification |
| 35–45 | Schedule follow-up and success metric |


## Success nudges

- Use one real deliverable, not a fake project
- Keep criteria under 10 items
- Ask for evidence at the start
- Do not accept until limitations are disclosed
- Use change requests rather than Slack side threads
