## Template philosophy

Templates should reduce blank-page friction. They should be specific enough to help but flexible enough to customize.

## Launch templates

| Template | Best for | Default criteria |
| --- | --- | --- |
| AI Research Report | Strategy, competitive analysis, market scans | Sources, freshness, claims matrix, risks, recommendation |
| AI Automation Delivery | Workflow automations, ops improvements | Demo, SOP, test evidence, rollback plan, owner handoff |
| Software Feature Delivery | Code/features shipped to a client | Requirements, tests, screenshots, deployment notes, limitations |
| Content Campaign Delivery | Marketing assets or campaigns | Assets, brand checks, approvals, usage rights, performance plan |
| Grant or Milestone Review | DAOs, accelerators, milestone-based work | Milestone criteria, evidence files, impact notes, reviewer decision |


## Template fields

Each template includes:

- Plain-language description
- Suggested acceptance criteria
- Required evidence types
- Common risks
- Reviewer checklist
- Example Mission Receipt wording

## Template quality bar

- [ ] Can a nontechnical client understand it in under 2 minutes
- [ ] Does it create objective criteria where possible
- [ ] Does it force limitations to be disclosed
- [ ] Does it avoid legal/compliance overclaims
- [ ] Does it support change requests
