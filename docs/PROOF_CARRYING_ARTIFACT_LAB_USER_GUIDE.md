# GoalOS Proof-Carrying Artifact Lab User Guide

This lab shows why GoalOS does not let every useful AI output become institutional memory.

## Core idea

A reusable artifact must be proof-carrying.

That means it needs:

```text
stable artifact identity
immutable version hash
ProofPacket history
baseline comparison
scope boundary
SelectionCertificate
EvolutionLedgerEntry
RolloutReceipt
RollbackReceipt
proof-backed upgrade right
```

## How to use the page

1. Open `proof-carrying-artifact-lab.html`.
2. Choose one artifact family.
3. Click **Run artifact gate**.
4. Watch four candidate versions move through the gate.
5. Inspect the tabs: Artifact vault, Proof history, Selection certificate, Evolution ledger, Rollout, Rollback, Upgrade right.
6. Download the synthetic demo package.

## What the demo teaches

```text
V0 rejected: persuasive output is not reusable capability.
V1 held: evidence exists, but baseline and scope are incomplete.
V2 canary: proof and rollback exist, but challenge window remains open.
V3 promoted: all hard gates clear within synthetic browser-demo scope.
```
