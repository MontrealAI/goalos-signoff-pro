# Mainnet and Token Boundary

This package is production-oriented for the mainstream Signoff workflow, but Mainnet and AGIALPHA functionality must remain gated.

## Allowed now

- Basic Signoff SaaS.
- Signed Mission Receipts.
- Private-beta Sepolia anchoring.
- Optional on-chain verification of hashes only.

## Not allowed yet

- Public Mainnet escrow.
- User-fund custody.
- AGIALPHA staking by public users.
- Automated payment settlement.
- Claims that the original 48-contract Mainnet system is production activated.

## Gates before Mainnet protocol economics

- Independent live revalidation PASS.
- Source identity reproducibility PASS.
- Contract remediation and audit.
- Production activation bounded canary.
- User-fund authorization for settlement.
- Public incident response and bug bounty.

## Messaging rule

Do not sell the token. Sell the workflow. AGIALPHA becomes useful when the workflow needs economic accountability.
