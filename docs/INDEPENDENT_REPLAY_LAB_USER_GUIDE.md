# Independent Replay Lab — User Guide

The lab demonstrates a simple GoalOS rule:

> A local pass is not public proof. A claim becomes review-ready only after replay.

## What to try

Open:

```text
independent-replay-lab.html
```

Choose one of the browser-local scenarios:

```text
Research
Software
Procurement
Safety
```

Click:

```text
Run replay council
```

You will see the replay gates advance:

```text
Manifest
Environment
Baselines
Runner config
ProofBundle
Replay logs
Validator reports
Cost ledger
Safety ledger
Claims matrix
Challenge window
Claim boundary
```

The browser shows a replay quorum, focused test count, claim level, and public evidence objects.

## What the lab teaches

GoalOS does not promote a claim because it sounds strong, because a local script passed, or because a single verifier said yes.

GoalOS promotes only a claim that survives independent replay, public-safe evidence, challenge windows, and claim boundaries.

## What it does not do

This public lab does not move value, verify real customers, connect a wallet, collect user data, certify a production system, or claim external audit.
