# Demo Lab v4 user guide

GoalOS Demo Lab lets a visitor understand the product without signing in, uploading files, connecting a wallet, or sending data.

## What to do

1. Open `demo-lab.html`.
2. Choose a demo mission.
3. Click **Launch proof cycle**.
4. Watch the gates advance: Commission, Submit, Map, Review, Accept, Receipt.
5. Open the sample Evidence Docket.
6. Verify the demo Mission Receipt.
7. Use `autonomous-demo.html` to run the GitHub Action that creates a downloadable demo bundle.

## What it proves

The demo does not certify real work. It demonstrates the product flow:

```text
Mission Contract → Evidence Docket → Human Review State → Mission Receipt
```

## What it does not do

```text
No sign-in
No upload
No wallet
No cookies
No analytics
No payment
No personal data
No confidential data
```
