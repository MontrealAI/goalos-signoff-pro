# GoalOS Process-Resolved Evidence Lab — User Guide

The lab demonstrates a simple GoalOS law:

```text
Final output is not proof.
Process-resolved evidence is proof-ready work.
```

Open:

```text
process-evidence-lab.html
```

Choose a synthetic scenario, then click:

```text
Run process pass
```

The browser-local demo shows four candidate AI work products:

```text
C0 — Final answer only: rejected.
C1 — Hidden tool path: quarantined.
C2 — Trace present, contradiction unresolved: held.
C3 — Process-resolved proof trace: decision-review ready.
```

The lab shows how GoalOS turns a final answer into an inspectable work surface:

```text
Mission Contract
→ Source Pass
→ Tool Scope
→ Claim Lineage
→ Contradiction Pass
→ Process Validator
→ Risk Ledger
→ Receipt Pointer
```

The important lesson is that GoalOS validates the path, not only the output.
