# Private-beta pilot checklist

## Before inviting users

- [ ] Supabase setup SQL completed on a fresh project
- [ ] Vercel deployment is green
- [ ] `/setup` shows all core services ready
- [ ] Site URL and exact Auth redirect URL configured
- [ ] One complete test Signoff succeeded with separate client, builder, and reviewer emails
- [ ] PDF and JSON receipts downloaded and verified
- [ ] A disposable receipt was revoked and displayed correctly
- [ ] Database backup policy documented
- [ ] Data retention and deletion policy documented
- [ ] Support contact and incident owner named
- [ ] Terms and privacy notice appropriate to the pilot published

## Pilot boundaries

- [ ] Invitation only
- [ ] Non-regulated, low-risk documents only
- [ ] No payment or escrow promise
- [ ] No blockchain-verification claim
- [ ] No autonomous factual-certification claim
- [ ] No more than 20 initial users
- [ ] Weekly review of errors, storage, and user feedback

## User interview questions

1. Was defining acceptance criteria easier than your current process?
2. Did evidence mapping reduce review time?
3. Did the client understand the difference between mechanical checks and human judgment?
4. Was the Mission Receipt useful outside GoalOS?
5. Would the user pay for this workflow, and at what frequency?
6. Which integration would remove the most friction?

## Initial success threshold

- 20 completed Signoffs
- 5 receipts shared outside the product
- At least 3 users request another Signoff
- No unresolved privacy or security incident
