Recommended architecture

Do not force all contracts into every Signoff.

Basic Signoff stays no-wallet and SaaS-simple. Verified Signoff anchors hashes. Secured Signoff adds bonded reviewers and challenge windows. Signoff & Settle adds escrow/reward rails only after production activation and user-fund authorization.

Tier 1

Signoff Basic

User sees: No wallet SaaS signoff. Create brief, collect evidence, review, receipt.

Protocol underneath: Protocol-compatible docket and receipt hashes; no public chain action.

$AGIALPHA: None required.

Signoff Verified

User sees: Verified receipt with anchored hashes and public verification page.

Protocol underneath: Brief/evidence/receipt commitments anchored after customer authorization.

$AGIALPHA: Sponsored protocol fee only if enabled.

Tier 3

Signoff Secured

User sees: Independent reviewer, challenge window, reputation and credential history.

Protocol underneath: Reviewer/evaluator bonds, attestations, challenges, and slashing readiness after audit.

$AGIALPHA: Reviewer/evaluator stake after activation.

Open tier
Tier 4

Signoff & Settle

User sees: Bounded proof-to-payment: accepted work releases reward; disputes pause settlement.

Protocol underneath: Job escrow, reward vault, dispute/appeal rails, treasury routing after user-fund authorization.

$AGIALPHA: Settlement, rewards, challenge stake, and protocol revenue when authorized.
