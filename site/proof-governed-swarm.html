PROOF-GOVERNED SWARM

A swarm becomes useful when proof decides what may propagate.

The key difference is not agent count. The key difference is whether claims are evidenced, validators are independent, risk is visible, rollback exists, and only accepted work enters memory.

Substrate

Compute · data · tasks · tools

Continuous inflows are useful only when scoped, metered, and bounded.

Coordination

Role graph · access graph · router

The coordinator selects who thinks, who acts, who verifies, when to stop, and when to escalate.

Proof

Trace root · output hash · claims matrix

Every accepted work unit must carry evidence sufficient for review and replay.

Governance

Risk gate · human gate · rollback

Score is advisory. Gates are mandatory.

Memory

Chronicle · capability package

Only proof-accepted experience influences future work.

Economy

Settlement-readiness signal

No value is moved in this public demo. The point is showing what would have to be proven before any economic consequence.

OPERATING PRINCIPLE
Maximum useful effect = learned coordination + formal verification + bounded tools + evidence memory + risk gates + human authority.
